# 🤝 CyberSH Code of Conduct

CyberSH is an open‑source AI framework built for hackers, devs, and researchers.  
We want a community that is transparent, respectful, and focused on building tools — not drama.

---

## 📜 Principles
- **Respect first** → No harassment, hate speech, or personal attacks.  
- **Hack, don’t harm** → Share exploits responsibly, never weaponize code.  
- **Transparency** → Be clear when reporting bugs or suggesting features.  
- **Collaboration** → Fork, patch, and PR with constructive intent.  
- **Zero tolerance** → Malicious contributions or spam will be removed.

---

## 🛡️ Moderation Rules
- Any **malicious code** or **exploit attempts** in PRs/issues will be deleted.  
- Repeat offenders may be **banned temporarily or permanently**.  
- Security reports must follow the **[VDP](SECURITY.md)** rules.  

---

## 🧩 Contribution Guidelines
- Keep commits clean and descriptive.  
- Respect repo structure — don’t dump random files.  
- Use manual testing when reporting bugs; automated spam will be rejected.  
- Only **official CyberSH forks** (starred by the team) are in scope for bug reports.  
- Modified forks or unrelated tools are **out of scope**.

---

## 🚀 Community Flow
1. Fork → Patch → Pull Request.  
2. Discuss ideas in Issues respectfully.  
3. Report vulnerabilities via **VDP** (see `SECURITY.md`).  
4. Earn recognition through valid contributions.  

---

## 🧠 Enforcement
- Maintainers have final say on moderation.  
- Violations = warning → temporary ban → permanent ban if repeated.  
- Valid bug reporters may earn a **CyberSH Security Researcher Certificate**.
